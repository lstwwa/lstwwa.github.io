文章目录

本身是打算鸽了这一篇文章，但是有几个朋友也想要有一个自己的网站，于是将方法放到这里，但是由于很多小伙伴使用的windows，但是最近windows11开始更新可能我文章都还没写完就被淘汰了，鉴于我个人使用的是macos于是写一个基于linux的教程。

写在开头

hexo是什么就不多说了，先说需要准备什么。

  • 你喜欢的linux，ubuntu或者什么都可以
  • 一些基础的github使用，当然稳定的链接也是很重要的
  • 一些基础的前端知识
  • 最重要的：遇到问题及时百度或者谷歌而不是垂头丧气

当然以上都可以没有，因为保姆级教程就是不需要动脑子。

弄一个操作系统

如果你已经有一个linux桌面版可以跳过这里。

这里以ubuntu举例，如果你很懒可以直接用ubuntu，这里推荐最新。

  • ubuntu下载链接

然后需要一个虚拟机，这里使用的是vmware，同样推荐vmware15

随后新建虚拟机一直到ubuntu安装，这里可以选择minimal安装，注意一定要包含桌面以及一个内置的firefox，如果使用的最新ubuntu就不用担心这个。

此时应该已经有一个ubuntu的桌面了

安装hexo

首先运行一遍更新一下源

sudo apt-get update

然后安装两件套

sudo apt-get install nodejs
sudo apt-get install npm

这里查看版本确定是否安装成功

nodejs -v
npm -v

到了这里就是安装hexo了，首先你需要自己准备一个文件夹，这里的文件夹是示范，你可以设置为blogxx或者xxx_blog这类型

mkdir hexo

提前进入自己创建好的文件夹

cd hexo

这里使用npm对hexo做全局安装

sudo npm install -g hexo-cli

如果没有报错这里就已经安装好了，随后初始化这个hexo文件夹，这时文件夹就有很多东西了

hexo init

本地测试

接下来就是hexo的几个常用命令

hexo clean    清楚hexo产生的临时文件
hexo g        编译hexo
hexo s        本地启用hexo
hexo d        上传hexo

最开始的测试可以使用这一条

hexo g && hexo s

随后可以在本地查看由hexo生成的第一个网页

http://localhost:4000/

如果这里可以看到生成的网站了那么恭喜，这里第一步已经完成了。

全球访问

这里需要连接到github，如果你无法连接可能是使用的移动网络，可以尝试使用电信或者联通即可解决问题，你也可以使用hosts来加快github的网络访问，这里不做讲解。

我们先安装hexo的git包

npm install hexo-deployer-git --save

首先注册一个github，这里是github网址：

https://github.com/

然后需要注册一个新的账号，请牢记自己的账号密码以及名字。

然后创建一个新的repository，这里需要名字+github.io

例如

lstwwa.github.io

下面什么都不要勾选，然后直接创建即可

随后你需要创建一个自己的token以便进行上传

点击【settings】

点击【Developer settings】

点击【Personal access tokens】后全部勾选并创建一个新的token以便后期上传，记得先复制出来

到这里已经完成了gitpage的配置

然后我们来到本地，打开命令行

git config --global user.email "email"
git config --global user.name "name"

然后输入来进行网站部署，这里名称输入名字，密码输入token

hexo d

如果这里上传成功了，我们来研究ssh免密码上传，干净的ubuntu首先创建一个ssh文件夹然后进去，如果有的话就可以不用创建

mkdir ~/.ssh
cd ~/.ssh

然后生成一个ssh，这里输入自己的邮箱，然后一路回车

ssh-keygen -t rsa -C "email"

然后查看自己的ssh，并且复制下来

cat id_rsa.pub

还是先进入【settings】

然后点击【SSH and GPG keys】

然后点击【New SSH Key】随后把ssh复制进去

然后来到命令行输入，然后输入yes，如果出现hi，你的名字吧啦吧啦就是成功了。

ssh -T git@github.com

这一切都就绪了，请一定保管好自己的token和自己的ssh。
如果要上传自己的网站了，可以使用

hexo clean && hexo g && hexo d

hexo文件目录树

hexo我们常用到的文件目录树如下

hexo
 ｜- _config.yml     --->config
 ｜- public          --->临时文件
 ｜- source          --->文章
             ｜- _posts
                         ｜- 文章
             ｜- CNAME     --->域名
 ｜- themes          --->主题文件
           ｜- 主题1
           ｜- 主题2

这里最重要的就是source中的几个文件夹，这里全是文章所以格外重要

其次CNAME文件中是写的本github.io用的个人域名，直接写上个人域名即可

日常的话经常用到的文件是_config.yml，里面包含了比较重要的一些配置

更好的文章生成器

项目地址：

https://github.com/Jamling/hexo-generator-index2

首先可以把整个博客备份一遍
随后先安装新的文章生成器然后再删除旧的

npm install hexo-generator-index2 --save
npm uninstall hexo-generator-index --save

随后在_config.yml添加以下内容

# 配置index2 generator，可以是数组或对象
index2_generator:
  per_page: 6
  order_by: -date
#  include:
#    - category Web # 只包含Web分类下的文章
  exclude:          # 不包含标签为Hexo的文章
    - category 大三下半期资料
    - tag hide

当然如果你不需要这个也没关系，因为这只是一个增强

写文章了

你要知道我们写的文章时xxx.md的文件，下图是一个标准的md开头（我定的），我们写的大部分文件都是post，也有一些页面会使用page。

---
layout: post
title: 文章名字
date: YYYY-MM-DD
tags: 
- 默认
categories: 
- YYYY-MM
- 默认
---
说明此篇文章
<!-- more -->

随后你得知道我们需要用到markdown格式来写文章，这是一种可以百度到的格式所以这里就不多说。下面是常用的一些写法

这里的标题需要你知道一般的markdown文章中只有一个大标题，随后才是其他小标题，他的格式是这样的：

#      大标题，通常只有一个
##     二级标题
###
####
#####
###### 六级标题，也可用于正文，不常用

其他的类型

- 备注
-- 二级备注
> 引用
>> 二级引用，这部分再一些主题中可以套娃
*内容* 表示强调
**内容** 加粗
<u>内容</u> 下划线

从这里的下划线可以看出，在这里的markdown中是可以使用网页的写法

[内容](链接)

这个是网页中的链接使用方法。

|名称|理论成绩|实验成绩|
| :-: | :-: | :-: |
|数字逻辑电路|理论：83|实验：82|

表格 是这样写的，中间:-:表示居中，这里需要看自己的主题是否支持。

代码块就是一个TAB后加上内容即可

拥有了这些知识基本就可以写一个自己的文章了，当然还有很多高级的markdown写法

文章置顶

在这个目录

node_modules/hexo-generator-index/lib/generator.js

如果你使用了index2，就在这个目录

node_modules/hexo-generator-index2/lib/generator2.js

然后添加中间这一部分代码

var config = hexo.config;
var posts = locals.posts.sort(generator.order_by);

//添加下面

    posts.data = posts.data.sort(function(a, b) {
        if(a.top && b.top) { 
            if(a.top == b.top) return b.date - a.date; 
            else return b.top - a.top; 
        }
        else if(a.top && !b.top) { 
            return -1;
        }
        else if(!a.top && b.top) {
            return 1;
        }
        else return b.date - a.date; 
    });

//添加上面 

var paginationDir = config.pagination_dir || 'page';
var path = generator.path || '';

然后再写文章时可以添加top来置顶

top: 1

短链接

npm install hexo-abbrlink --save

permalink:  :abbrlink.html

在_config.yml中添加

abbrlink:
  alg: crc32  #算法，支持crc16(默认) 和crc32，crc16的url比较短
  rep: hex    #进制 dec(十进制，默认) 和hex（十六进制）
